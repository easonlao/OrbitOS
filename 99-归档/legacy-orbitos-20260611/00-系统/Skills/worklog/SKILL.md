# 工作日志管理工作流

## 可用内部工具

| 工具 | 用途 |
|------|------|
| `create_worklog()` | 生成今日工作日志 |
| `append_worklog(content)` | 智能追加内容到今日日志 |
| `rebuild_worklog_index()` | 重建日志索引 |

## 日志存储位置

`{VAULT}/02-日记/工作日志/YYYYMMDD_工作日志_周X.md`

## 日志模板结构

```markdown
# YYYY-MM-DD 周X 工作日志

## 今日重点
## 今日Todo
## Git 提交
## 重点记录
## 关键决策
## 问题与风险
## 明日计划
```

## 内容追加分类规则

| 关键词/特征 | 目标章节 |
|-------------|----------|
| 问题、bug、报错、异常、风险 | 问题与风险 |
| 完成、实现、开发、重点、修复、上线 | 重点记录 |
| 决策、选择、方案、拍板 | 关键决策 |
| 今天、重点、目标 | 今日重点 |
| 待办、todo、跟进、下一步候选 | 今日Todo |
| 明天、计划、明日 | 明日计划 |
| 其他 | 重点记录 |

## Agent 记录要求

- Agent 自动写入默认进入 `## 重点记录`。
- 结构化条目应包含 `记录方`、`写入方式`、`会话ID`、`来源目录`。
- `关键决策` 和 `明日计划` 未经用户确认时，应明确标注为候选或待确认。
