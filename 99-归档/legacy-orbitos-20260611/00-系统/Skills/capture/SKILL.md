# Capture — 对话价值捕获

## OBJECTIVE

从对话中捕获涌现的价值。不是预先计划的研究，而是聊着聊着发现"这个值得记下来"。

**适用场景：**
- 讨论中出现可复用的知识点
- 推理过程中发现有价值的逻辑框架
- 模糊的想法逐渐清晰，值得标记为研究方向
- 决策过程本身值得记录（不只是结果）

**不适用场景：**
- 处理网页/视频内容 → 用 `knowledge`
- 有目的的深度调研 → 用 `research`
- 记录今天做了什么 → 用 `worklog`

## WORKFLOW

### Step 1: 识别内容类型

从对话上下文中判断捕获内容属于哪一类：

| 类型 | 特征 | 输出位置 |
|------|------|---------|
| 知识点 | 可复用的概念、定义、原理 | `03-知识/<主题>/` (type: card) |
| 逻辑框架 | 方法论、决策模型、思维模式 | `03-知识/<主题>/` (type: note) |
| 研究方向 | 还没深入但值得后续探索的 | `01-收件箱/待整理/` (type: note, status: draft) |
| 决策过程 | 推理链、权衡分析、最终判断 | `02-日记/反思/` (type: reflection) |
| 行动项 | 明确的下一步 | `02-日记/工作日志/` 追加到今日重点记录 |

### Step 2: 提取 + 结构化

从对话中提取核心内容，生成带 frontmatter 的 markdown：

```yaml
---
title: "<从对话中提炼的标题>"
type: card              # card|note|reflection
topic: <topic>          # ai|dev|reading|work|project|tools|writing|life|system
workspace: "<目标工作区>"
created: "YYYY-MM-DD HH:MM:SS"
modified: "YYYY-MM-DD HH:MM:SS"
tags: ["capture", ...]
source: manual
status: active
---
```

**提取原则：**
- 只提取核心观点，不要复述对话原文
- 保留原始推理链的关键步骤
- 用 `[[wikilink]]` 关联已有笔记
- 中文输出

### Step 3: 确认

向用户展示捕获内容：

```
## 沉淀内容

**类型:** 知识点 / 逻辑框架 / 研究方向 / 决策过程
**位置:** 03-知识/<主题>/<标题>.md

### 内容预览
[提取的markdown内容]

确认保存？或告诉我修改。
```

用户确认后写入文件。用户说"不用了"则取消。

## 规则

- 不保存对话原文，只提取有价值的部分
- 一个 capture 只保存一个独立的知识点/框架，不要合并多个
- 不确定类型时，优先归为知识点（card）
- 保存后在当日工作日志追加记录：写入 `## 重点记录`，并注明记录方、写入方式以及捕获了什么
