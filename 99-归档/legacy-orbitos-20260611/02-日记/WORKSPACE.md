# 02-日记 工作区规范

## 用途

记录每日状态、工作日志、反思、周期复盘和 LifeOS 人际事件。

## 模块边界

- `每日/`：原始日常记录，偏时间线，不承担项目总结和长期知识沉淀。
- `工作日志/`：执行事实、完成事项、关键过程、自动化记录的主时间线。
- `反思/`：对事实的解释、感受、模式识别、认知更新。
- `复盘/`：按周/月/阶段对多个记录源做总结，不记录碎片过程。
- `人际事件/原始记录/`：尽量只写事实，不做过早结论。
- `人际事件/事件/`：允许在事实基础上形成分析、判断、策略。

## 允许

- 每日记录
- 工作日志
- 反思
- 周报和月报
- 工作日志索引 `工作日志/INDEX.md`
- 人际事件
- 关系复盘与事件原始记录

## 禁止

- 外部网页原文
- 长期课程资源
- 项目资产文件

## 命名

每日记录使用 `YYYYMMDD_每日.md`。工作日志使用 `YYYYMMDD_工作日志_主题.md`，常规每日工作日志可使用 `YYYYMMDD_工作日志_周X.md`。反思使用 `YYYYMMDD_反思主题.md`。人际事件使用 `YYYYMMDD_事件主题.md`，旧 LifeOS 文件可保留 `event_YYYYMMDD_主题.md` 作为迁移 trace。

`02-日记` 只保留一层主题目录：`每日/`、`工作日志/`、`反思/`、`复盘/`、`人际事件/`。不得再创建 `worklog/YYYY/MM` 这类年月嵌套；需要按时间检索时更新 `工作日志/INDEX.md` 或交给 Agent 生成索引。`人际事件/` 允许内部保留 `事件/` 与 `原始记录/` 两类目录，作为 LifeOS 子系统边界。

历史兼容层目录 `02-日记/todos/` 已被完全移除，不在此一级目录结构内。`02-日记` 的唯一 canonical Todo 位置是工作日志内的 `## 今日Todo` 章节。


## Frontmatter

`workspace` 必须为 `02-日记`，`type` 使用 `worklog`、`reflection`、`review`、`event` 或 `note`。其中 `人际事件/原始记录/` 默认仍使用 `event`，通过目录边界区分“原始事实”与“分析事件”，不再新增 `event-raw` 类型。

## 协作边界

### 用户主导

- `今日重点` 的目标和主观判断。
- 反思中的真实感受、动机、价值判断。
- 人际关系中的立场、解释和结论。
- 是否接受某条“明日计划”或“下一步”。

### Agent 可自动写入

- Git 提交、文件产出、工具执行摘要。
- 已经完成的事实性记录。
- 经用户明确说出的 Todo 整理。
- 周/月复盘的草稿整理。

### Agent 不应擅自写入

- 替用户虚构反思。
- 把候选建议写成已确认决策。
- 把解释性判断写成既成事实。
- 创建或读写已被废弃的 `todos/` 目录下的独立文件。


## 工作日志标准结构

工作日志新文件必须使用以下章节：

1. `## 今日重点`
2. `## 今日Todo`
3. `## 重点记录`
4. `## 关键决策`
5. `## 问题与风险`
6. `## 明日计划`
7. `## Git 提交`

其中：

- `Git 提交` 由 Hook/自动化写入。
- `重点记录` 是 Agent 自动追加记录的默认 section。
- `关键决策` 只记录已确认决策。
- `明日计划` 未经用户确认时应显式标注“候选”。

## Agent 追责要求

所有由 Agent 或自动化写入 `工作日志/` 的条目，必须能追溯以下信息：

- 谁写的（`actor_id`）
- 什么时候写的（`timestamp`）
- 通过什么机制写的（`recorded_via`）
- 来自哪个工作目录或上下文（`origin_cwd`）
- 属于哪个会话（`session_id`，取不到时写 `unknown`）

`重点记录` 中的 Agent 条目应优先使用统一结构化格式，而不是自由文本散写。

## 子 Skill

使用 `workspace-journal`。

## 领域 Skill

- `worklog`：工作日志、Git 提交、Agent 产出。
- `reflect`：反思。
- `review`：周/月复盘。
- `lifeos`：人际事件、人物画像、关系复盘和事件决策。

## LifeOS 规则

- 人际事件放在 `人际事件/事件/`。
- 原始事件记录放在 `人际事件/原始记录/`。
- 人物档案不放在日记区，放在 `05-资源/人物档案/people.json`。
- 不再向 `space/crafted/lifeos` 写入新内容。
