# systemd 服务管理方案与操作边界

## 背景

Hermes 清理了当前机器上的 user 服务（之前有 12 个，现清理至 9 个）。清理过程中发现：Hermes 能直接管理 `systemctl --user` 服务（无需 sudo），但需建立标准化操作流程。

## 对齐结论

### 管理范围

仅限 user 服务（`systemctl --user`），系统服务（需要 sudo 的）暂不纳入。

### 操作流程

1. Hermes 先调查现状，给出操作建议及影响说明
2. 经用户确认后方可执行
3. 执行后记录留档

### 当前 user 服务清单（9 个）

运行中（3）：
- `dbus.service` — D-Bus 用户会话总线
- `gpg-agent.service` — GPG 加密代理
- `hermes-gateway.service` — Hermes 网关

已停（6，无害）：
- `dirmngr.service` — GPG 网络证书管理
- `keyboxd.service` — GPG 公钥管理
- `launchpadlib-cache-clean.service` — Launchpad 缓存清理
- `pk-debconf-helper.service` — debconf 通信
- `snapd.session-agent.service` — snap 用户会话代理
- `systemd-remount-fs.service` — 已 mask

### 已清理（3）

- `hermes-web-ui.service` — 文件已删除
- `surfsense-proxy.service` — 文件已删除
- `at-spi-dbus-bus.service` — 已停止
