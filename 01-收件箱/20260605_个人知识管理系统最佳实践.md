# 个人知识管理系统最佳实践

## 核心功能

一个有效的 PKM 系统必须可靠地完成4项工作：

1. **捕获 (Capture)**：快速记录想法、灵感、信息
2. **组织 (Organize)**：将零散内容结构化，建立分类体系
3. **检索 (Retrieve)**：在需要时快速找到相关信息
4. **连接 (Connect)**：发现不同知识点之间的关联

> 关键洞察：大多数人优化的是捕获和组织阶段，但检索和连接才是价值所在。捕获是容易的部分，连接是困难的部分。

## 三种主流方法论

### PARA

由 Tiago Forte 提出，将信息分为四个领域：

- **P**rojects（项目）：有明确目标和截止日期的任务
- **A**reas（领域）：需要长期维护的责任范围
- **R**esources（资源）：感兴趣的主题资料
- **A**rchives（归档）：已完成或不再活跃的内容

**优点**：结构清晰，易于执行
**适用场景**：任务导向型用户

### Zettelkasten（卡片盒笔记法）

由德国社会学家 Niklas Luhmann 发明：

- 每条笔记是一个原子化的知识点
- 通过双向链接建立笔记之间的关联
- 强调"连接"而非"分类"
- 知识在连接中涌现

**优点**：促进创造性思维，知识网络自然生长
**适用场景**：研究型、思考型用户

### BASB（Building a Second Brain）

Tiago Forte 的系统化方法：

- **CODE** 框架：Capture → Organize → Distill → Express
- 强调"渐进式总结"（Progressive Summarization）
- 知识最终要通过"表达"产生价值

**优点**：完整的从输入到输出的工作流
**适用场景**：内容创作者、知识工作者

## 最佳实践原则

1. **少即是多**：不要过度分类，保持简单
2. **定期回顾**：每周/每月回顾，清理和连接
3. **渐进式总结**：不要一次性整理完，分层提炼
4. **输出驱动**：以写作为目标组织知识
5. **工具服务于流程**：先确定工作流，再选择工具

## 与 OrbitOS 的关联

OrbitOS 的设计融合了多种方法论的优点：

- 文件夹结构参考了 PARA 的领域划分
- 双向链接体现了 Zettelkasten 的连接理念
- 技能系统支持 CODE 工作流的实现
