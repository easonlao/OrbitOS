# 仓位管理方法论

## 概述

仓位管理（Position Sizing）回答交易中最基础也最容易被忽略的问题：**每次下注多少**。

它是唯一不需要预测市场就能直接改变账户收益曲线的工具。同样的入场和出场，不同的仓位管理，长期收益可以差数倍甚至数十倍。

仓位管理可以按维度分为四大类：基础模型、波动率调整、进阶策略、组合管理。另有一个视角来自 PYTA/ICT：它看似不讨论仓位，但止损位置、入场时机、分批止盈本质上都是仓位决策。

## 核心问题

仓位管理要解决三个子问题：

| 问题 | 含义 | 对应决策 |
|------|------|---------|
| 暴露多少 | 单笔交易承担多少风险 | 风险百分比、固定分数、凯利比例 |
| 何时变 | 仓位应该什么时候增加或减少 | 马丁格尔/反马丁格尔、金字塔、波动率调整 |
| 怎么分 | 多品种/多策略时如何分配 | 相关性分配、风险平价、等权重 |

## 一、基础模型

### 固定数量法（Fixed Lot）

每笔交易使用相同数量的合约/股数。

- **优点**：极度简单，零计算
- **致命缺陷**：不考虑账户规模变化。账户从 10 万亏到 5 万，还开同样的仓位，实际上风险翻倍
- **适用**：几乎不推荐单独使用

### 固定分数法（Fixed Fractional）

每笔交易风险为账户资金的固定百分比（通常 1-2%）。

```
头寸规模 = (账户资金 × 风险百分比) / (止损距离 × 点值)
```

**示例**：账户 10 万，风险 2%，止损 50 点，点值 100 元/点 → 最大风险 2000 元 → 开 0.4 手

- **优点**：自适应账户规模，绝对风险始终可控
- **缺点**：赚钱时加仓慢，亏钱后恢复也慢
- **适用**：所有散户的基础底线。应该先掌握这个方法，再考虑更高级的

固定 2% 和随机仓位（1-5% 之间随意开）的回测对比——同一策略，前者收益 +23%、回撤 -11%，后者收益 +5%、回撤 -29%。

### 凯利公式（Kelly Criterion）

数学上最大化长期增长率的最优下注比例：

```
f* = (b·p - q) / b
```
- p = 胜率，q = 败率（1-p），b = 盈亏比

**核心困境**：凯利公式把胜率和盈亏比当作两个独立参数，但实盘中它们被第三个变量**时间（持有时间）**锚定。

- 持有时间短 → 暴露少 → 胜率高 → 但单笔利润薄（高频、做市商）
- 持有时间长 → 暴露多 → 被扫止损概率上升 → 胜率低 → 但趋势走出来时利润丰厚（趋势波段）

散户站在长时间端：20-50% 胜率区间，盈亏比底线 2:1。入场质量可以在三角内部滑动位置（好的入场把胜率从 30% 拉到 40%），但不改变三角结构本身。

**散户用法**：凯利公式的第一个功能不是告诉你开多少——是告诉你策略有没有正期望。胜率 35% + 盈亏比 2:1 才刚过正期望线，大部分散户的策略根本达不到。f* 为负 → 不是仓位问题，是策略问题。

用四分之一凯利 + 硬上限 5%，取置信下界而非均值算参数。

### Optimal f（Ralph Vince）

Ralph Vince 提出的改进版凯利。在不同风险暴露水平下回测，找到最大化几何收益的那个 f 值。

- 比凯利更灵活，基于实际损益分布而非简单的胜率/盈亏比
- 但对历史数据的过拟合风险更高
- 需要手动回测确定 f 值

### 风险百分比法 vs 凯利公式

| 维度 | 风险百分比法 | 凯利公式 |
|------|------------|---------|
| 输入 | 只关心止损距离 | 需要胜率、盈亏比（都不稳定） |
| 保守度 | 用户自定（1-2%） | 数学导出，但参数难准 |
| 底层约束 | 无额外假设 | 假设胜率和盈亏比独立（实际被时间锚定） |
| 风险 | 人为控制，永远不过度 | 参数高估时过度暴露 |
| 正确角色 | 底线层，必须遵守 | 筛子层，验证期望 + 参考上限 |
| 推荐人群 | 新手、散户、所有基础层 | 有稳定回测数据的进阶者 |

## 二、波动率调整类

### ATR 头寸单位法（海龟法则）

根据市场波动率（ATR）动态调整头寸，确保每笔交易的波动风险一致。

```
头寸单位 = 账户资金 × 1% / ATR(N)
```

- 波动大的品种自动减仓，波动小的品种自动加仓
- 所有品种的"日波动风险"被标准化
- 详见 [[头寸管理]]

### Volatility Targeting

设定目标波动率（如年化 15%），根据近期实际波动率反向调整仓位：

```
仓位 = 目标波动率 / 当前实际波动率
```

- 实际波动率 30% → 仓位减半到 50%
- 实际波动率 10% → 仓位加到 150%
- 广泛应用于对冲基金和风险平价策略

## 三、进阶策略类

### 马丁格尔（Martingale）

亏损后加倍下注，直到盈利一次收回所有损失并获得初始赌注的利润。

**在交易中的应用**：
- 价格每跌一定幅度，加倍买入
- 不设止损，等待价格回归
- 表面逻辑：只要不出现连续极端走势，最终一定盈利

**致命缺陷**：
- 连续 10 次亏损后，仓位膨胀 1024 倍，散户资金不可承受
- 一次极端行情即可清空账户
- 外汇市场 EU 杠杆上限 30 倍，加密 2 倍，直接限制操作空间

**结论**：金融交易中几乎不具备长期可行性。本质是"用无限资金换有限利润"，散户没有无限资金。

详见 [[马丁格尔与反马丁格尔]]

### 反马丁格尔（Anti-Martingale）

与马丁格尔完全相反：**盈利后加码，亏损后减码**。

**在交易中的应用**：
- 获利后增加仓位（顺势而为）
- 亏损后减少仓位（保护本金）
- 比例不必是 2 倍，可以用 1.3-1.5 倍渐进

**优势**：
- 亏损时仓位自动收缩，锁死最大回撤
- 盈利时仓位扩大，顺着趋势推高收益
- 根本上改变了"赚小亏大"的散户通病

**核心挑战**：需要判断何时是"趋势"—反复加仓的仓位在回调中会遭受更大打击。

详见 [[马丁格尔与反马丁格尔]]

### 金字塔加仓

在已有盈利仓位基础上，沿趋势方向追加头寸。

**三种变体**：
- **正金字塔**：初始仓位最大，每次加仓递减（保守）
- **倒金字塔**：初始仓位最小，每次加仓递增（激进）
- **等额加仓**：每次加仓相同规模（均衡）

**关键原则**：
- 只在盈利的头寸上加仓，不为亏损头寸加仓
- 加仓后的综合止损必须保证整体不亏损
- 加仓频率和间隔必须有规则

## 四、组合管理类

### 相关性仓位分配

当同时持有多个品种时，不能单独计算每个品种的仓位——必须考虑它们之间的相关性。

**核心原理**：
- 两个高度正相关品种（如 EUR/USD 和 GBP/USD）相当于重仓同一方向
- 按品种分别算 2% 风险，但如果它们相关系数 0.8，实际组合风险远超 2%
- 总风险 = √(各品种风险的平方和 + 2×相关系数×品种A风险×品种B风险)

**简化做法**：
- 高相关品种（>0.7）视为同一方向，合并计算风险敞口
- 低相关/负相关品种可以分散但不要过度分散
- 散户组合不超过 3-5 个低相关品种

详见 [[相关性仓位分配]]

### 风险平价（Risk Parity）

使组合中每个资产/策略的**风险贡献相等**，而非资金分配相等。

**传统等权重**：A 类波动 10%，B 类波动 30% → 两者各投 50% 资金 → B 贡献 90% 风险

**风险平价**：调整资金分配，使 A 和 B 的风险贡献相等 → A 投更多资金，B 投更少

**桥水全天候策略的核心思想**：通过风险平价，在任何经济环境中组合都有保护的资产类别。

**散户适配**：需要杠杆才能把低波动资产的预期收益拉到可接受水平，散户不容易做到。概念有价值，实操门槛高。

详见 [[风险平价]]

## 五、PYTA/ICT 中的仓位管理逻辑

PYTA 从不明确讨论"仓位管理"，但它的每一条规则本质上是仓位决策：

### 1. 止损位置 = 风险暴露决策

PYTA 要求止损设在流动性池外、扫流动性影线外。这个"位置选择"决定了止损距离，进而决定了每笔的风险暴露。

**散户常见的错误**：先决定开多大仓位，再随便放一个止损。PYTA 的逻辑是反过来的——先看合理的止损在哪，再用止损距离反算仓位大小。

### 2. 入场时机 = 仓位时机

Kill Zone 内的入场不是随意行为——它限制了你只能在特定时间窗口暴露仓位。在正常时间（亚洲盘期间）不加仓，在 Kill Zone 外不建新仓。这本质上是用**时间维度**约束仓位创建时机。

### 3. 分批止盈 = 结构化退出

PYTA 的标准做法：目标 1 走 50%，目标 2 走 50%。这本质上是仓位退出策略——既不是全部一口吃完，也不是无限持有等待回调。

- 目标 1 减半仓 → 锁定了部分利润，降低剩余仓位的心理压力
- 目标 2 清仓 → 在深层流动性池获利了结

### 4. 流动性陷阱与仓位

PYTA 核心教训之一是：不要在流动性猎取阶段建仓。翻译成仓位语言——最差的建仓时机是在价格朝你预期的反方向运行、散户在恐慌性平仓的时候。因为这时候你无法判断这是一种猎取（会反转）还是真突破（会继续）。

### 5. 做市商库存管理 vs 散户仓位管理

做市商通过调整报价（买价/卖价的偏移）来管理库存，散户通过调整仓位大小来管理风险。两者的目标相同：**在风险可控的前提下最大化盈利**。方法不同，但底层是一个问题。

## 仓位管理的两层架构

### 全局层（总敞口天花板，始终运行）

```
账户总资金 × 总敞口上限（≤15%）→ 总风险预算
各活跃持仓当前止损风险 → 已占用额度（动态，非静态）
可用额度 = 总风险预算 - 已占用额度
```

| 参数 | 建议值 | 原理 |
|------|--------|------|
| 总敞口上限 | ≤ 15% | 亏 10% 需赚 11%，亏 25% 需 33%——差距指数放大 |
| 单品种上限 | 5% | 单一标的极端行情不过多暴露 |
| 标的数量 | 2-3 个低相关 | 钱不多时分散只分散收益，不分散风险 |

止损移到保本位 → 该笔占用 → 0，自动释放额度。部分止盈 → 剩余仓位按新止损重算。平仓 → 释放。

### 局部层（单笔交易生命周期）

```
信号 → 确定止损/止盈 → 盈亏比 ≥ 2:1？
              ↓ 是
        风险百分比法算仓位 → 向全局请求可用额度
              ↓
    实际仓位 = min(局部算的, 全局给的够, 方向集中度约束)
              ↓
    开仓 → 持有 → 止损移动 / 目标1减半 / 目标2清仓
              ↓
    结果记录 → 更新胜率统计 → 反馈给全局层
```

单笔参数独立闭环，但受全局额度约束。两个层通过"请求额度 → 返回可用"双向交互。

**单笔内部的进一步拆分**：如果不在单一价格推满，可以用分档进场策略在单笔内部设多档权重（进取/均衡/回撤）。不管分几档，这笔交易占用的风险预算是固定的。详见 [[分档进场策略]]。
