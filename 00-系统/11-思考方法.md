# 思考方法

OrbitOS 的思考方法不是要你学习一套术语，而是让你在复杂任务开始前，看见 Agent 准备怎样把问题想清楚。

简单任务不需要选择。只有问题目标模糊、需要取舍、要做复杂规划，或准备提交重要结论时，Agent 才会先给出一张小选择卡。

## 你会看到什么

例如你说“我想做一个长期项目，但不知道从哪里开始”，Agent 不会立刻写一大段方案，而会先说：

```text
这一步需要先确定分析方式：

1. 反向推导
   从最终想达成的结果倒推阶段和第一步。

2. 5W1H
   先把目标、对象、范围和约束补全，避免一开始漏掉关键问题。

推荐：1，因为你已经有方向，主要缺少推进路径。
回复 1 / 2；也可以说“直接做”或描述你的想法。
```

你不需要记住方法名。回复数字、说“从结果倒推”“换几个角度看看”，甚至说“直接做”都可以。

## 显式启动

如果你不想等 Agent 判断，也可以直接输入：

```text
启动思考模式
```

这句话会让 Agent 先执行一次很短的选择流程：回看当前讨论已经确认什么、你真正卡在哪里、这一步处于什么阶段，再给出适合当前情况的选择卡。它不会直接开始分析，也不会默认替你选定一种方法。

你也可以把当前要梳理的任务一起说清楚：

```text
启动思考模式：帮我梳理这个项目为什么卡住，以及下一步应该怎么推进。
```

这时 Agent 会根据当前讨论和这句补充，先给出候选与推荐理由，等你确认后才开始。

## 八种方法分别帮什么忙

| 当你遇到的情况           | Agent 可能推荐 | 它帮你得到什么          |
| ----------------- | ---------- | ---------------- |
| 事情很模糊，不知道还缺什么信息   | 5W1H       | 一张完整的问题地图与待确认项   |
| 结论听起来对，但担心逻辑不扎实   | 苏格拉底提问     | 隐含假设、证据缺口与替代解释   |
| 在犹豫一件事值不值得启动      | SWOT 分析    | 优势、短板、机会与风险的判断依据 |
| 想摆脱惯例，从根本重想一条长期路线 | 第一性原理      | 底层事实与重构后的选择      |
| 目标很大，但不知道第一步做什么   | 反向推导       | 从终点倒推到当前的最小行动    |
| 已经想清楚，但讲不清、写不顺    | 金字塔原理      | 一句核心结论和清晰的支撑结构   |
| 讨论一直卡在同一种视角       | 六顶思考帽      | 把事实、风险、创意和感受分开看  |
| 方案准备提交，想主动找盲点     | 批判性思维      | 前提风险、逻辑断点与补强动作   |

## 怎么选才不会有压力

你不必“选对”。选择只是决定这一轮优先从哪个角度开始：

- 想先把事情看全：选 `5W1H`。
- 想把遥远目标变成行动：选 `反向推导`。
- 想判断要不要做：选 `SWOT 分析`。
- 想检查方案有没有漏洞：选 `批判性思维`。
- 想把已有内容讲清楚：选 `金字塔原理`。

如果仍然不确定，直接说“你推荐哪一个，为什么”，Agent 应用一句简单理由帮你决定，而不是把八种方法都推给你。

## 选择之后会怎样

选择只对当前任务有效。Agent 会按选定角度推进，并在完成时用一句话说明：

```text
本轮思考方式：反向推导。作用：把长期目标拆成当前可执行的第一步。
```

它不会把选择自动写成你的长期偏好，也不会因此新建项目、知识卡或其他文件。你说“直接做”时，Agent 会跳过选择卡，按任务本身继续处理。

## 两种方法可以配合，但不会堆叠

复杂任务偶尔会组合两种方法，例如先用 `5W1H` 看全问题，再用 `反向推导` 找到行动路径。Agent 最多应建议两种，并说明各自作用；不会把全部八种方法套到同一件事上。
