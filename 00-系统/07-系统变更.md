# 系统变更

## 当前版本：v0.1.0

OrbitOS 已具备可以实际使用的基础框架。完整技术历史保存在 `.orbitos/CHANGELOG.md`，本页只保留用户需要知道的变化。

## 当前能力

- 根 `AGENTS.md` 是所有 Agent 的统一使用入口。
- [[../02-时间线/今日|今日]] 是日常 Dashboard，待确认和下一步有独立入口。
- 收件箱支持盘点、确认、已入库原件和知识草稿流程。
- 项目状态、机器 event、当前知识和时间线视图已经分开。
- 不同 Agent 可以登记部署位置、运行限制、经验和踩坑。
- Validation 可以检查 schema、目录、链接、event 命名和入库记录。
- Hindsight Bridge 已完成独立测试 bank、API 与 single-bank MCP 的试点验证。
- Runtime 与 Product Repo 已分离，系统更新通过普通 Git pull 进入 Runtime。

## 最近调整

- `00-系统/` 重构为按顺序阅读的用户说明书。
- 收件箱固定输入入口改为 `00-粘贴.md`，保证稳定显示在最前面。
- OrbitOS 开发项目改为根层状态文件、`docs/` 专项记录和 `repo/` 产品仓库。
- `.orbitos/AGENTS.md` 改为中文，并继续作为唯一正式开发规则源。
- 当前 OrbitOS 不再依赖旧版 Skills；Role、Thinking Mode 和新 Skills 设计留待后续阶段。
- 普通对话不要求结构化；实质性工作结束后由脚本生成最小完成记录，再由 Validation 检查持久化结果。

## 尚未完成

- 收件箱完整自动化整理仍在逐步验证。
- Agent Role、Thinking Mode 和 Skills 资产尚未设计落地。
- Hindsight 正式主 bank 的命名和迁移策略仍待确认。
- 文档一致性与 Vault Audit 仍会继续加强。
