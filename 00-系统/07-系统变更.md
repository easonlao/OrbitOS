# 系统变更

## 当前版本：v0.1.0

OrbitOS 已具备可实际使用的基础框架。本页只保留“使用者需要知道什么变了”，不记录完整开发过程。

## 现在可以用什么

- 根 `AGENTS.md` 是所有 Agent 的统一使用入口。
- `02-时间线/今日.md` 是日常 Dashboard，事项和入口集中在这里。
- 收件箱支持盘点、确认、已入库原件和知识草稿流程。
- 项目状态、机器 event、当前知识和时间线视图已经分开。
- 不同 Agent 可以登记部署位置、运行限制与自己的工作档案。
- Validation 会检查关键结构、链接和边界问题。
- Runtime 与 Product Repo 已分离：发布内容在 `repo/`，运行态内容留在本地 Runtime。

## 最近值得知道的变化

- `00-系统/` 重构为按顺序阅读的用户说明书。
- 收件箱固定输入入口改为 `00-粘贴.md`，保证稳定显示在最前面。
- Agent 启动时默认只读最小必要上下文，经验与踩坑改为按需展开。
- Agent 现在会按任务定位目标区域，再读取对应的局部 `AGENTS.md`、workflow 和 rule。
- 机器运行层与人读设计层已明确分开，减少了混杂和重复说明。
- 知识冲突现在单独走收敛流程：冲突中的两份提炼结果先退回 `已入库` 作为索引源，再由一份冲突说明草稿收敛成唯一最终知识。
- Agent 之间的交接现在有单独入口：`agents/BOARD.md` 只放交接索引，`agents/handoff/` 只放 handoff 模板和归档，交接需要先经审核把关。
- M4 多 Agent 稳定协作已经收口：通过并发私有写入、handoff 审核闭环、事件边界和多 Agent 证据验证。

## 尚未完成

- 收件箱完整自动化整理仍在逐步验证。
- Hindsight 正式主 bank 的命名和迁移策略仍在收口中。
- Vault Audit 仍会继续加强。
