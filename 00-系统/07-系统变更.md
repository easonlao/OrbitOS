# 系统变更

## 当前版本：v0.1.0

OrbitOS 已具备可实际使用的基础框架。本页只保留“使用者需要知道什么变了”，不记录完整开发过程。

## 现在可以用什么

- 根 `AGENTS.md` 是所有 Agent 的统一使用入口。
- `02-时间线/今日.md` 是日常 Dashboard，事项和入口集中在这里。
- 收件箱支持盘点、确认、已入库原件和知识草稿流程。
- 项目状态、机器 event、当前知识和时间线视图已经分开。
- 不同 Agent 可以登记部署位置、运行限制与自己的工作档案。
- Validation 会检查关键结构、链接和边界问题。
- Runtime 与 Product Repo 已分离：发布内容在 `repo/`，运行态内容留在本地 Runtime。

## 最近值得知道的变化

- `00-系统/` 重构为按顺序阅读的用户说明书。
- 收件箱固定输入入口改为 `00-粘贴.md`，保证稳定显示在最前面。
- Agent 启动时默认只读最小必要上下文，经验与踩坑改为按需展开。
- Agent 现在会按任务定位目标区域，再读取对应的局部 `AGENTS.md`、workflow 和 rule。
- 机器运行层与人读设计层已明确分开，减少了混杂和重复说明。
- 知识冲突现在单独走收敛流程：冲突中的两份提炼结果先退回 `已入库` 作为索引源，再由一份冲突说明草稿收敛成唯一最终知识。
- Agent 之间的交接现在有单独入口：`agents/BOARD.md` 只放交接索引，`00-系统/agents/handoff/` 只放 handoff 实例和归档，模板源统一收口到 `.orbitos/templates/00-系统/agents/handoff/TEMPLATE.md`。
- M4 多 Agent 稳定协作已经收口：通过并发私有写入、handoff 审核闭环、事件边界和多 Agent 证据验证。
- Agent 入口现在允许按需读取 `00-系统/08-本地协作偏好.md`，把当前 Runtime 的个人合作方式与 Product Repo 的系统规则分开。
- 本地协作偏好现在有通用模板：首次初始化会从 `.orbitos/templates/00-系统/08-本地协作偏好.TEMPLATE.md` 生成本地副本，Startup Sync 看到模板标记时会提醒用户补成自己的版本。
- Hermes 融入后的共享记忆边界已经进一步收紧：Hindsight 继续只保留最小稳定摘要，不承接项目状态、待确认事项、完整 handoff 或临时草稿。
- Hindsight 入口现在明确默认走直连 HTTP API，不要求额外 MCP 或 hooks；根 `AGENTS.md` 已补上最小 `stats / tags / recall` 命令与 memory 触发门。
- Hindsight 相关表述已再次去冗余：入口、rule 和 workflow 现在只保留必要执行说明，避免把扩展适配能力误写成系统已自动接线。
- 根 `AGENTS.md` 已进一步瘦身：保留入口骨架与路由，执行模式细则、Hindsight 命令与 retain 边界已下沉回 workflow / rule，减少新 Agent 首屏负担。
- Hindsight 正式接入已确定 bank 策略：正式记忆库使用 `orbit-os`，保留 `orbitos-test` 作为测试 bank，旧记忆不再作为 OrbitOS 新规范的迁移基线。
- Hindsight 协议已吸收官方社区反馈：OrbitOS 当前默认继续优先 tags 过滤、稀疏 retain 和非强依赖 `reflect`，不把宽召回或高频反思直接写成默认策略。
- Hindsight runtime 已从单一 pilot 校验升级为双层 bank 规则：`write_event.py` 与 `run-validation.py` 现已同时接受 `orbitos-test` 与 `orbit-os`，测试样例继续固定留在测试库，正式长期记忆可合法写入正式库。
- Hindsight 现已补上第一条正式 retain 的生产样板：正式库优先从“已有本地主源、已多轮验证、会持续影响 Agent 行为”的协作原则或稳定环境事实开始，而不是直接写当日状态或测试样例。
- Hindsight 正式主库 `orbit-os` 已完成首条真实写入，并已用 `stats / tags / recall` 验证不是空接线；当前正式记忆链已从“样板可写”进入“正式可用”状态。
- 运行层不再维护独立 ADR 机制：相关 core rule、入口路由与用户说明已移除；项目级决策历史由各项目自己的工程文档约定处理，避免 Runtime 再承担第二套决策记录系统。
- 运行层校验已去掉对独立 `02-时间线/待确认.md` 的旧路径依赖；当前时间线口径只保留 `今日.md` 内的“当前待确认 / 可继续”区块。
- 研究型 handoff 的确认收口边界也已收紧：未确认前仍需显式写清待确认点与二次核查，但一旦达到 `C3 + confirmed`，研究链可以自然收口，不再需要为了形式保留伪待确认状态。
- 思考模块现已作为正式思考方法层接入：8 种模式一次性保留，但只作为分析、handoff 与自检时可按需调用的方法，不作为新的强制流程系统。
- OrbitOS 正在补执行生命周期硬约束：Agent 在真正动手前需要先区分 `direct / delegated / confirm_first`，复杂任务不再允许默认压成单 Agent 直做。
- OrbitOS 也开始区分 `automatic / manual_supported / concept_only` 三种真实执行状态，避免把“需要手动触发”或“只有概念设计”的东西误写成系统已自动具备的能力。

## 尚未完成

- 收件箱完整自动化整理仍在逐步验证。
- Hindsight 正式主 bank 已可写可读，当前剩余的是持续实测与后续准入策略收紧。
- Vault Audit 仍会继续加强。
