# 系统变更

## 当前版本：v0.1.0

OrbitOS 已具备可以实际使用的基础框架。完整技术历史保存在 `.orbitos/CHANGELOG.md`，本页只保留用户需要知道的变化。

## 当前能力

- 根 `AGENTS.md` 是所有 Agent 的统一使用入口。
- [[../02-时间线/今日|今日]] 是日常 Dashboard，待确认和下一步有独立入口。
- 收件箱支持盘点、确认、已入库原件和知识草稿流程。
- 项目状态、机器 event、当前知识和时间线视图已经分开。
- 不同 Agent 可以登记部署位置、运行限制、经验和踩坑。
- Validation 可以检查 schema、目录、链接、旧目录编号、关键文档边界、event 命名和入库记录。
- `init-runtime.py` 现在会在 Git metadata 可用时维护本地 `.git/info/exclude`，让 agent 私有工作目录保持运行态隔离，而不污染共享仓库 ignore 规则。
- 项目经验与 agent 经验已正式分层：项目特有踩坑优先留在项目 `docs/LESSONS-LEARNED.md`，跨项目经验才进入 agent profile。
- Agent 档案已拆成“轻量 profile + 按需经验文件”：启动只读必要上下文，完整经验与踩坑按需展开，降低冷启动 token 消耗。
- Hindsight Bridge 已完成独立测试 bank、API 与 single-bank MCP 的试点验证。
- Runtime 与 Product Repo 已分离，系统更新通过普通 Git pull 进入 Runtime。

## 最近调整

- `00-系统/` 重构为按顺序阅读的用户说明书。
- 收件箱固定输入入口改为 `00-粘贴.md`，保证稳定显示在最前面。
- OrbitOS 开发项目改为根层状态文件、`docs/` 专项记录和 `repo/` 产品仓库。
- `.orbitos/AGENTS.md` 改为中文，并继续作为唯一正式开发规则源。
- 当前 OrbitOS 不再依赖旧版 Skills；Role、Thinking Mode 和新 Skills 设计留待后续阶段。
- 普通对话不要求结构化；实质性工作结束后由脚本生成最小完成记录，再由 Validation 检查持久化结果。
- Runtime 私有 agent 目录优先走本地 `.git/info/exclude`，共享 `.gitignore` 只保留跨 runtime 的稳定本地内容边界。
- Startup Sync 不再默认读取完整经验与踩坑；已注册 agent 先读轻量 profile，需要时再按经验入口渐进式披露。
- Startup Sync 不再固定读取用户说明书及全部时间线页面；默认只读 registry、轻量 profile、环境报告和今日摘要。
- Agent 现在会按任务语义定位工作区域，并沿目标路径读取项目或仓库的局部 `AGENTS.md`，不再只依赖根入口猜测工作边界。
- 根 `AGENTS.md` 已收缩为最小入口契约：只保留必做流程、任务路由、暂停条件和固定约束，具体步骤统一下沉到 workflow 与 rule。
- ADR 已明确为 Product Repo 内的决策历史层：Architecture/Design 记录当前状态，ADR 保存重大选择的原因与取舍；Runtime 不维护第二份系统 ADR。
- MAP、README 与 AGENTS 的职责已统一：MAP 只导航；README 只服务可独立发布或使用的对象，不作为内部项目默认文件；AGENTS 保存 Agent 必需的目标、架构、规则与路由。
- 项目规划数据流已统一：event 保存操作证据，STATUS 保存详细状态与任务，ROADMAP 显示目标和总体状态，今日只显示当日变化，CHANGELOG 只记录已发布变化。
- 项目公共规则已进入全局按需路由；具体项目只需补充自己的目标、架构和特殊边界，不再重复项目目录与状态规则。
- ROADMAP 使用完成条件清单显示进度：已验证条件标记完成日期，执行小任务仍由 STATUS 管理。
- 用户无需主动触发项目工作流：Agent 自动区分当场修改、跨会话事项和长期目标；路线与当前优先级仍由用户确认。
