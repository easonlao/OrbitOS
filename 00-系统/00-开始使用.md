# 开始使用 OrbitOS

OrbitOS 是你与 Agent 持续协作的工作台：材料不会散落，任务可以接着推进，重要结论有地方可回看。

你只需要做三件事：把材料放进来、用自然语言提出任务、在关键时刻确认方向。

## 第一次怎么开始

1. 用 Obsidian 打开 OrbitOS 文件夹。
2. 把文字、链接或资料放进 `01-收件箱/`；随手内容可写入 `01-收件箱/00-粘贴.md`。
3. 对 Agent 说：

```text
请先看看收件箱里有什么，给我处理建议，不要移动文件。
```

4. 日常查看 `02-时间线/今日.md`，了解现在做到哪里、什么需要你决定、下一步可以做什么。

## 你可以让 OrbitOS 做什么

| 你的需要 | 你可以怎么说 | 你会得到什么 |
|---|---|---|
| 整理材料 | `整理收件箱，先给建议。` | 材料主题、去向建议与需要确认的事项 |
| 推进一件事 | `继续推进这个项目，先告诉我现状。` | 当前进展、阻塞与可继续动作 |
| 沉淀结论 | `把这份材料整理成知识草稿。` | 可确认的知识草稿，不会直接定稿 |
| 看全局节奏 | `同步进度。` | 今日页中的当前进展与入口 |
| 深入阅读 | `这份材料适合进入阅读吗？` | 是否适合阅读模块的建议与后续选择 |

## 重要原则

- 不确定时先让 Agent 给建议，不要让它直接处理。
- 涉及移动、删除、归档或定稿时，由你最后确认。
- 不需要记住命令；描述你想要的结果即可。
