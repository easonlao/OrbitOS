# 开始使用 OrbitOS

OrbitOS 是一个让你与多个 Agent 在同一套文件中持续协作的工作台。

你通过 Obsidian 阅读和确认内容；Agent 负责执行任务、维护状态和留下可追溯记录。更换 Agent 或设备后，工作仍然可以从文件中的记录继续。

## 第一次使用

1. 用 Obsidian 打开 OrbitOS 根目录。
2. 先运行 `python .orbitos/scripts/init-runtime.py`，用它初始化你当前使用的工作副本；它会创建本地的 `01-收件箱/00-粘贴.md`、`02-时间线/今日.md` 和 `02-时间线/本周.md`。
3. 把文字、链接或待整理材料放入 `01-收件箱/00-粘贴.md`，也可以直接放进 `01-收件箱/`。
4. 与 Agent 对话，让它从根目录 `AGENTS.md` 开始。
5. Agent 完成 [[06-术语表#Startup Sync|Startup Sync]] 后，会告诉你今日页可见的事项。
6. 日常优先打开 `02-时间线/今日.md`，不需要翻阅机器文件。
7. 如果需要交给另一位 Agent 继续，先看 `00-系统/agents/BOARD.md`。

这里说的“当前工作副本”，就是你实际在 Obsidian 中使用的这一份 OrbitOS；旧文档里如果偶尔出现 Runtime，也指的是同一个对象。

## 第一次对 Agent 说什么

如果你想最稳妥地开始，第一轮对话直接用下面这段就够了：

```text
请先阅读 AGENTS.md 并执行 Startup Sync。
然后盘点一下收件箱，先给建议，不要移动文件。
```

这段话的作用很简单：

- 第一行让 Agent 按 OrbitOS 的入口和边界接入，而不是自由发挥。
- 第二行让它先做最小动作，只盘点、不移动，避免第一次使用就把目录弄乱。

如果你想启用动态人物模块，可以在首次使用后明确对 Agent 说“请启用动态人物模块，走人物基线初始化流程”。Agent 会按 24 题、5 档倾向的基线问卷生成 `00-系统/09-人物档案.md`，再把协作相关的派生段落写入本地协作偏好。当前这套问卷仍然只是 MBTI 种子框架，用来低成本定调，不是最终版正式问卷。问卷阶段默认一次只问 1 题，并直接使用固定题面，不应该由 Agent 自行发挥题意。

如果后续你发现 Agent 对你的协作方式判断不准，可以再直接说：

- `请根据最近的真实行为校准人物模块。`
- `请把这条校准建议正式写回人物档案。`

前一句只会生成建议，后一句才会在你确认后真正更新主源。

当你需要交接给下一位 Agent 时，只要记住一个最小原则：

- 研究中的内容先走 `研究型 handoff`，明确证据、推断和待确认点。
- 已开始执行或准备收口的内容走 `执行型 handoff`，明确已完成、未完成和接手动作。
- 如果问题本身比较复杂，Agent 也可以按任务特征自动推荐不同思考方法，比如先全维扫描、再倒推拆解、最后做提交前自检；这是一层思考方法，不是额外流程，你也可以直接指定要用哪一种。

不需要自己发明交接格式，Agent 会按现有模板和 workflow 来写。

首次接入一个尚未注册的 Agent 时，可以直接说：

```text
你现在接入 OrbitOS，请先阅读 AGENTS.md 并执行 Startup Sync。
如果还没有注册，先告诉我需要确认的接入信息，不要直接写入系统。
```

## 平时怎么用

- **有材料**：丢进收件箱。
- **有任务**：直接和 Agent 说自然语言，不需要记命令。
- **想看进度**：打开 `02-时间线/今日.md`，或者对 Agent 说“同步进度”。
- **需要决策**：看 `02-时间线/今日.md`。
- **不知道做什么**：看 `02-时间线/今日.md` 的可继续入口。
- **回顾一周**：查看 `02-时间线/本周.md`。
- **想找长期结论**：先看 `04-知识/` 或项目文档；只有需要跨会话快速召回时，才让 Agent 用 Hindsight 补充。Hindsight 适合最小稳定摘要，不适合直接当项目进度、待确认事项或 handoff 的长期存放处。

## 最常见的 5 类请求

1. `整理一下收件箱，先给建议，不要移动。`
2. `继续推进某个项目，先告诉我当前做到哪里。`
3. `把这批已入库内容整理成知识草稿。`
4. `请同步进度。`
5. `请启用动态人物模块，走人物基线初始化流程。`

如果你不确定一句话会不会让 Agent 扩大范围，就在末尾补一句：`先不要移动文件`、`先不要定稿`、`先告诉我再改`。

1. [[00-开始使用|开始使用]]：先完成第一次协作。
2. [[01-目录说明|目录说明]]：知道每个区域放什么。
3. [[02-日常协作|日常协作]]：了解每天如何与 Agent 工作。

5. [[04-Agent协作|Agent 协作]]：了解多个 Agent 如何接入和积累经验。
6. [[05-安全与边界|安全与边界]]：了解哪些动作需要你确认。
7. [[06-术语表|术语表]]：按需查询，不需要预先阅读或记忆。
8. [[07-系统变更|系统变更]]：查看当前版本的重要变化。

## 你不需要主动阅读的内容

- `.orbitos/`：Agent 的规则、工作流、校验和运行记录。
- event、queue、schema：用于追溯和校验，不是日常阅读入口。
- `repo/`：只有开发 OrbitOS 本身时才需要进入。
