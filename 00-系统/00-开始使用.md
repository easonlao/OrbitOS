# 开始使用 OrbitOS

OrbitOS 是一个让你与多个 Agent 在同一套文件中持续协作的工作台。

你通过 Obsidian 阅读和确认内容；Agent 负责执行任务、维护状态和留下可追溯记录。更换 Agent 或设备后，工作仍然可以从文件中的记录继续。

## 第一次使用

1. 用 Obsidian 打开 OrbitOS 根目录。
2. 先运行 `python .orbitos/scripts/init-runtime.py`，它会创建本地的 `01-收件箱/00-粘贴.md`、`02-时间线/今日.md` 和 `02-时间线/本周.md`。
3. 把文字、链接或待整理材料放入 `01-收件箱/00-粘贴.md`，也可以直接放进 `01-收件箱/`。
4. 与 Agent 对话，让它从根目录 `AGENTS.md` 开始。
5. Agent 完成 [[06-术语表#Startup Sync|Startup Sync]] 后，会告诉你今日页可见的事项。
6. 日常优先打开 `02-时间线/今日.md`，不需要翻阅机器文件。
7. 如果需要交给另一位 Agent 继续，先看 `00-系统/agents/BOARD.md`。

首次接入一个尚未注册的 Agent 时，可以直接说：

```text
你现在接入 OrbitOS，请先阅读 AGENTS.md 并执行 Startup Sync。
如果还没有注册，先告诉我需要确认的接入信息，不要直接写入系统。
```

## 平时怎么用

- **有材料**：丢进收件箱。
- **有任务**：直接和 Agent 说自然语言，不需要记命令。
- **想看进度**：打开 `02-时间线/今日.md`，或者对 Agent 说“同步进度”。
- **需要决策**：看 `02-时间线/今日.md`。
- **不知道做什么**：看 `02-时间线/今日.md` 的可继续入口。
- **回顾一周**：查看 `02-时间线/本周.md`。

## 你不需要主动阅读的内容

- `.orbitos/`：Agent 的规则、工作流、校验和运行记录。
- event、queue、schema：用于追溯和校验，不是日常阅读入口。
- `repo/`：只有开发 OrbitOS 本身时才需要进入。
