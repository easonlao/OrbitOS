# 日常协作

你不需要记住复杂命令。日常只需要自然地提出任务，OrbitOS 负责让 Agent 先读取状态、完成工作并同步结果。

## 开始一次对话

Agent 进入 OrbitOS 后会先执行 [[06-术语表#Startup Sync|Startup Sync]]：

1. 确认自己是谁、部署在哪里。
2. 先读取自己的轻量档案；完整经验和踩坑只在任务命中或返工时按需展开。
3. 读取今日页，必要时再读项目状态。
4. 向你输出一段简短摘要。

Startup Sync 只读取状态，不会擅自推进任务。

如果当前工作副本里已经存在一份本地固定合作方式，Agent 在任务命中时再读取那份本地协作偏好；这类约定帮助 Agent 更贴近你的习惯，但不自动视为所有工作副本通用的系统规则。若其中还有动态人物档案投影出来的段落，Agent 也只能把它们当作当前工作副本的本地实例来参考。首次初始化生成的模板会带有提醒标记，Agent 看到后应先提醒你补成自己的版本。

## 提出任务

你可以直接说：

```text
整理一下收件箱里最近新增的内容，先给建议，不要移动。
继续推进 OrbitOS 项目，先告诉我当前做到哪里。
把这次修复记录下来并同步进度。
今天做了什么？还有什么需要我确认？
```

指令越简短，Agent 越应该执行最小、可回退的动作；不应自行扩大到其他项目或目录。

如果任务看起来需要长期推进，但当前没有合适项目，Agent 会先判断是否应该挂接到已有项目；只有在确实需要单独立项时，才会先说明理由并询问你是否创建新项目。

## 查看今日页

`02-时间线/今日.md` 是主要 [[06-术语表#Dashboard|Dashboard]]，集中展示：

- 今天发生的关键变化。
- 需要你确认的事项。
- 可以继续推进的入口。
- 收件箱、项目和 Agent 的重要状态。

今日不是永久历史。跨日完成项会进入项目状态、本周回顾或机器事件记录，不应一直堆在今日。

## 完成工作

Agent 完成实质性工作后会执行 [[06-术语表#Progress Sync|Progress Sync]]：

1. 把本次结果编译成一条最小完成记录，不要求你使用固定话术。
2. 运行校验。
3. 先更新正确的状态源，例如项目 `STATUS.md`。
4. 再刷新今日里的摘要、需要确认的事项和可继续入口。
5. 如果出现可复用经验，记录到自己的 Agent 档案。

你也可以随时说“同步”“同步进度”或“更新进度”触发刷新。

如果这次工作要交给另一位 Agent 继续，请先按 `.orbitos/templates/00-系统/agents/handoff/TEMPLATE.md` 生成 handoff 实例，并放到 `00-系统/agents/handoff/`，先经审核把关，再让接手方阅读最后确认。

普通讨论和只读查询不会为了“留痕”制造额外文件；只有产生持久化结果时才进入这套收尾流程。

## 思考模式怎么用

OrbitOS 还有一层轻量的 [[06-术语表#思考模式|思考模式]]。它不是新的强制流程，也不是额外角色，而是 Agent 在复杂任务里用来分析、拆解、表达和自检的方法层。

你可以这样理解：

- 不指定时，Agent 可以按任务特征自动推荐一种方法，并显式告诉你本轮用了什么。
- 你也可以直接指定，例如“先用全维扫描看一遍”“这次用倒推拆第一步”“收尾前做一次提交前自检”。
- 如果任务本身很直接，Agent 不需要为了形式硬套思考模式。

思考模式的作用是帮助这轮任务想清楚，不会自动生成新的文档、项目或知识卡片。

## 每周回顾

`02-时间线/本周.md` 只展示当前周的重要时间线、方向、风险和下周建议。

它由 Weekly Review 根据 [[06-术语表#Event|Event]] 和项目状态生成，不由普通 Progress Sync 每次追加。进入新的一周时，旧内容保存到 `02-时间线/归档/`。
