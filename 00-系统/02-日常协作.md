# 日常协作

你不需要记住复杂命令。日常只需要自然地提出任务，OrbitOS 负责让 Agent 先读取状态、完成工作并同步结果。

## 开始一次对话

Agent 进入 OrbitOS 后会先执行 Startup Sync：

1. 确认自己是谁、部署在哪里。
2. 读取自己的经验和踩坑。
3. 读取系统当前状态、待确认和下一步。
4. 如果涉及具体项目，再读取项目状态。
5. 向你输出一段简短状态摘要。

Startup Sync 只读取状态，不会擅自推进任务。

## 提出任务

你可以直接说：

```text
整理一下收件箱里最近新增的内容，先给建议，不要移动。
继续推进 OrbitOS 项目，先告诉我当前做到哪里。
把这次修复记录下来并同步进度。
今天做了什么？还有什么需要我确认？
```

指令越简短，Agent 越应该执行最小、可回退的动作；不应自行扩大到其他项目或目录。

## 查看当前状态

[[../02-时间线/今日|今日]] 是主要 Dashboard，集中展示：

- 今天发生的关键变化。
- 当前需要你确认的事项。
- 可以继续推进的下一步。
- 收件箱、项目和 Agent 的重要状态。

今日不是永久历史。跨日完成项会进入项目状态、本周回顾或机器事件记录，不应一直堆在今日。

## 完成工作

Agent 完成实质性工作后会执行 Progress Sync：

1. 记录本次实际做了什么。
2. 运行校验。
3. 先更新正确的状态源，例如项目 `STATUS.md`。
4. 再刷新今日、待确认和下一步。
5. 如果出现可复用经验，记录到自己的 Agent 档案。

你也可以随时说“同步”“同步进度”或“更新进度”触发刷新。

## 每周回顾

[[../02-时间线/本周|本周]] 只展示当前周的重要时间线、方向、风险和下周建议。

它由 Weekly Review 根据 event 和项目状态生成，不由普通 Progress Sync 每次追加。进入新的一周时，旧内容保存到 `02-时间线/归档/`。
