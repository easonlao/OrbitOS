# 你的确认与内容安全

OrbitOS 的原则是：Agent 可以推进工作，但不会替你决定内容的去留或结论的定稿。

## 这些事情会先问你

| 场景 | 你会看到什么 |
|---|---|
| 移动、删除或归档材料 | 涉及哪些内容、会去哪里、为什么这样处理 |
| 把草稿变成正式知识或成品 | 候选结果与等待确认的入口 |
| 新旧结论互相冲突 | 冲突点与可选处理方式 |
| 新建项目或扩大处理范围 | 这样做的理由与影响 |
| 设置自动任务 | 它会做什么、何时运行、是否会改动内容 |

## 你可以怎样控制动作

- `先给建议，不要修改。`
- `先写草稿，不要定稿。`
- `只看这个项目，不要扩到其他目录。`
- `先告诉我影响，再执行。`
- `发现问题就报告，不要自动修复。`

当系统完成一次写入型工作时，会检查结构与链接是否正常；如果发现问题，应该如实报告，而不是把失败伪装成完成。

你的收件箱、项目、知识、时间线和个人偏好都属于当前工作副本。更新 OrbitOS 的系统功能不会把这些内容当成公共资料覆盖掉。
