# 模块与扩展

OrbitOS 的核心已经能处理日常工作：接住材料、推进项目、沉淀知识、让你在今日页看见进展。

当某一种工作开始变得重要，再打开相应模块即可。每一页都从“它解决什么问题、何时适合用、如何开始”来说明，不要求你理解实现细节。

| 你正在面对的情况 | 对应说明 |
|---|---|
| 想把读书或长文阅读变成持续过程 | [[模块/阅读|阅读模块]] |
| 想让 Agent 更理解自己的协作偏好 | [[模块/人物|人物模块]] |
| 想减少跨会话时重复说明背景 | [[模块/Hindsight|Hindsight 记忆模块]] |
| 正在管理有代码与发布需求的项目 | [[模块/工程|工程模块]] |
| 想把重复检查或回顾交给系统 | [[模块/自动化|自动化模块]] |
| 需要多个 Agent 分工或接力 | [[模块/协作|多 Agent 协作模块]] |

## 思考方法一直都在

思考方法是核心能力，不是模块。遇到复杂、模糊或需要权衡的事情，Agent 会先用一张简单选择卡告诉你有哪些思考路径，以及它推荐哪一种。你可以选编号、说出自己的想法，或者直接说“直接做”。详细说明见 [[11-思考方法|思考方法]]。

它的作用是让你看见这次任务会怎样被想清楚，而不是给系统增加一层负担。

## 共通原则

- 先判断是否真的需要，再决定是否启用。
- Agent 应先解释会带来什么变化，不能因为你问了一句就直接安装、建目录或创建任务。
- 暂停使用一项能力，不等于删除已经存在的用户内容。
