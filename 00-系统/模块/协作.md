# 多 Agent 协作模块

多 Agent 协作模块让一项工作可以被不同 Agent 安全接力，而不是靠你在聊天窗口之间反复转述上下文。

## 什么时候需要它

- 一个 Agent 做完调研，另一位 Agent 要继续实现或审核。
- 一项任务需要不同角色分工，例如设计、实现和审查。
- 当前任务会跨会话或跨设备持续，且接手者需要理解关键判断。

简单的一问一答、一次性小修改不需要进入协作流程。只有真的需要接力时，才建立一份正式交接记录。

## 交接会保留什么

一份好的交接记录不是完整聊天记录，而是让下一位 Agent 能立刻判断：

- 已经完成了什么。
- 结论基于什么证据。
- 还有什么尚未确认或需要复核。
- 接手后最小的下一步是什么。

项目 `STATUS.md` 仍然只记录项目现状；handoff 只记录协作链和接手动作，两者不互相替代。

## 如何开始

```text
交给另一位 Agent 继续。
```

接手时，另一位 Agent 只需要说“获取交接工作”。你不需要自己维护格式，但仍应确认关键决策和高风险动作。
