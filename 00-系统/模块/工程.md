# 工程模块

工程模块面向有代码、测试和发布需求的项目。它让项目进展、代码和发布内容各自留在清楚的位置，方便你随时知道项目做到哪里，而不会把讨论和临时材料混进代码仓库。

## 项目里你会看到什么

初始化后的代码项目有三个可见入口：

- **项目说明**：项目要解决什么问题、当前范围和重要背景。
- **项目状态**：现在做到哪里、卡在哪里、下一步是什么。日常优先看这里。
- **`repo/`**：代码、测试、发布说明和版本记录。需要查看实现或发布内容时再进入。

项目中的 `AGENTS.md` 是 Agent 进入该项目时会读取的说明。你通常不需要维护它；如果项目需要接入新的 Agent，直接让 Agent 从项目入口开始即可。

## 什么时候适合用

当一个项目有自己的 `repo/`，并且你需要持续经历需求澄清、规划、实现、验证和发布时，就适合使用工程模块。

## Agent 会怎样推进

Agent 会按项目所处阶段推进需求澄清、规划、实现、验证和发布。涉及代码项目时，它会推荐合适的 Matt Pocock Skills，并说明这项 Skill 会帮助完成什么；你不需要预先记住技能名或安装顺序。

每完成一个关键阶段，Agent 应把进展、验证结果和下一步写回项目状态。这样即使换 Agent 或隔一段时间再继续，你也能先从项目状态了解情况。

## 如何开始

```text
我要新建一个代码项目。请先判断是否需要立项；如果需要，按工程项目的方式初始化，并告诉我之后应从哪里查看项目状态和代码。
```

或对已有项目说：

```text
请检查这个项目是否已经具备代码项目的最小入口和边界，先只报告缺口，不要修改。
```
