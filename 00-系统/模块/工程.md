# 工程模块

工程模块面向有代码、测试和发布需求的项目。它让 Agent 清楚区分“项目怎么管理”和“产品仓库里写什么”，避免把状态、讨论和临时材料混进代码仓库。

## 什么时候适合用

当一个项目有自己的 `repo/`，并且你需要持续经历需求澄清、规划、实现、验证和发布时，就适合使用工程模块。

代码型项目至少应保留三个清晰入口：

- 项目 `AGENTS.md`：Agent 在项目里从哪里开始、遵守什么边界。
- 项目 `STATUS.md`：项目现在做到哪里、卡在哪里。
- `repo/`：产品代码、测试、发布说明和版本控制所在的位置。

项目状态留在项目层，代码与发布物留在 `repo/`。这使得你既能管理项目，又不会污染产品仓库。

## 与 Matt Pocock Skills 的关系

工程模块会把 Matt Pocock Skills 作为推荐的工程推进方式：在合适阶段显式调用对应技能，完成初始化、需求澄清、规划、实现和审查。

它不要求你一次安装全部 Skills。Agent 应先根据当前阶段说明建议使用哪一个，以及如果不使用它，项目该如何保留等价的产物和验证。

## 如何开始

```text
我要新建一个代码项目。请先判断是否需要立项；如果需要，按工程项目的方式初始化，并告诉我建议使用哪些 Matt Pocock Skills。
```

或对已有项目说：

```text
请检查这个项目是否已经具备代码项目的最小入口和边界，先只报告缺口，不要修改。
```
