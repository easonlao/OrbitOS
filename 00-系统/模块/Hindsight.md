# Hindsight 记忆模块

Hindsight 用来减少跨会话协作中的重复解释：让 Agent 在合适的时候找回少量已经稳定、以后还会有用的事实和经验。

## 什么时候适合用

- 你经常换 Agent 或隔很久再继续同一类工作。
- 有稳定的环境信息、协作习惯或长期经验，反复说明很浪费时间。
- 你希望 Agent 能先回忆已有共识，再开始讨论。

## 它不替代什么

Hindsight 不是你的知识库、项目管理器或日记。

- 项目当前做到哪里，仍然看项目 `STATUS.md`。
- 已确认的知识结论，仍然留在知识区。
- 今天需要决定什么，仍然看今日页。
- 临时猜测、未确认想法和完整对话，不应该被当成长期记忆。

这样做是为了让记忆保持轻，而不是变成另一个难以维护的信息堆。

## 如何开始

```text
我想减少跨会话时重复说明背景。请评估 Hindsight 是否适合我，并说明它会记住什么、不记住什么。
```

如果你确认启用，Agent 会先说明需要的连接与配置。启用后，你也可以随时明确要求：

```text
这是一条稳定的长期协作原则；请确认本地主源后再考虑写入记忆。
```

Agent 应保留本地文件作为可追溯主源，不能只因为记忆系统“记得”就当作事实已落盘。
