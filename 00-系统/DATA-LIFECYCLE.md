# 数据生命周期

这页只回答一个问题：你把东西放进 OrbitOS 后，它会在 Obsidian 里怎么流动。

## 一句话

```text
先丢进收件箱 -> agent 盘点并写到时间线 -> 你确认去向 -> 进入项目/知识/资源/输出 -> 不再常用时归档
```

## 你每天主要看哪里

优先看 [[../02-时间线/今日|今日]]。

`今日.md` 是当前 Dashboard，应该告诉你：

- 最近做了什么
- 有什么需要你确认
- 下一步可以做什么
- 收件箱有没有待处理内容
- 项目或系统有没有状态变化

如果你只想知道 OrbitOS 现在怎么样，先看这里，不需要去翻内部文件。

## 1. 收件箱：先放进来

你临时想到的内容、网址、图片、旧笔记、复制来的材料，都可以先放进 `01-收件箱/`。

收件箱可以很乱，不要求格式。

`01-收件箱/粘贴.md` 是自由输入入口，不需要 frontmatter。你可以直接把内容贴进去。

agent 处理收件箱时，第一步只做盘点：

- 看里面大概有哪些主题
- 标出可能的去向
- 写出为什么这样判断
- 把需要你确认的内容放到 [[../02-时间线/待确认|待确认]]
- 把可继续的动作放到 [[../02-时间线/下一步|下一步]]

第一轮不会直接移动、删除，也不会自动做成知识卡片。

## 2. 时间线：先让你看见状态

时间线不是资料库，而是当前状态入口。

主要文件：

- [[../02-时间线/今日|今日]]：今天和当前状态
- [[../02-时间线/本周|本周]]：方向性洞察，不是流水账
- [[../02-时间线/待确认|待确认]]：需要你确认的事项
- [[../02-时间线/下一步|下一步]]：可以继续推进的入口

`02-时间线/归档/` 只用于旧时间线视图的快照，例如过期的 `今日.md` 或阶段性时间线页面。它不是全局归档，也不用于存放项目、知识、资源或原始输入。

agent 完成实质性工作后，应该刷新相关时间线，让你不用追问也能知道当前进度。

你说“同步”“同步进度”“更新进度”时，也会触发这一步。

## 3. 确认：决定东西去哪里

OrbitOS 不应该把所有内容都自动沉淀。

以下内容需要你确认后，才能变成长期内容：

- 系统规则
- 架构决策
- 正式产物
- 知识卡片
- 从 `.orbitos/docs/` 提升到 `00-系统/` 的用户说明

待确认事项集中放在 [[../02-时间线/待确认|待确认]]，不要散落在各个文件里。

## 4. 项目：正在推进的事情

确认某件事属于一个具体项目后，进入 `03-项目/`。

项目可以包含代码、草稿、素材、中间状态和应用文件，不需要完全遵守 OrbitOS 的知识卡片结构。

但每个项目至少应该有：

- `README.md`：这个项目是什么
- `STATUS.md`：现在做到哪里、卡在哪里、下一步是什么

agent 处理项目时，应该先读项目自己的 `README.md` 和 `STATUS.md`，再继续工作。

## 5. 知识：确认过才沉淀

`04-知识/` 只放已经确认、可复用、值得长期保留的内容。

知识卡片宁少不多。不是所有收件箱内容、对话内容、项目过程都要进知识区。

适合进入知识区的内容通常是：

- 经过你确认的原则
- 可复用的方法
- 稳定的概念解释
- 已验证的经验总结

没有确认的内容，只能作为候选，不应该直接进入知识区。

## 6. 资源：被整理过的引用材料

`05-资源/` 放被处理过、未来可能会被引用的材料，例如图片、PDF、网页转存、视频笔记和外部资料索引。

资源区和收件箱的区别是：

- 收件箱是“先丢进来”
- 资源区是“已经知道它是什么，未来可以被引用”

资源本身不需要每个目录都维护 MAP。多数时候通过 Markdown 引用、双链或附件路径使用即可。

## 7. 输出：Obsidian 内的成品

`06-输出/` 只放在 Obsidian 内产出的 Markdown 成品，以及这些成品需要引用的图片、PDF 等资源。

项目代码、应用文件和工程中间状态不放在这里。

如果一个输出来自某个项目，可以在 Markdown 里链接项目，但不要把项目目录搬进输出区。

## 8. 归档：退出当前视野

归档表示“不再属于当前工作视野”，不是删除。

常见情况：

- 旧的时间线快照进入 `02-时间线/归档/`
- 已完成或暂停很久的完整对象进入 `99-归档/`
- 旧系统设计或迁移记录保留到内部历史位置

是否归档由具体对象决定，不需要为了归档而归档。

## 机器记录在哪里

你在 Obsidian 里主要看可见 Markdown。

agent 还会在 `.orbitos/` 里写机器记录，用来追踪“谁、什么时候、因为什么、做了什么”。

你平时不需要阅读这些文件。它们的作用是让不同 agent 接入时能追溯状态，而不是替代你的 Obsidian 阅读入口。

## 不会自动发生的事

OrbitOS 默认不会：

- 把收件箱内容直接变成知识卡片
- 把对话内容直接提升为系统规则
- 把未确认内容写进正式产物
- 因为用了 Hindsight 就认为内容已经进入 OrbitOS
- 静默移动或删除你的文件

如果 agent 认为某件事应该沉淀，会先放到待确认或下一步。
