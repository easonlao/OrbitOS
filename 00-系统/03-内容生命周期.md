# 内容生命周期

一份材料进入 OrbitOS 后，不会立刻被当成知识或正式成果。它会先保留原貌，再由你确认应该去哪里。

```text
收件箱 -> 处理建议 -> 你的确认 -> 项目 / 知识草稿 / 阅读 -> 长期回看或归档
```

## 收件箱：先接住，不急着决定

文字、网址、图片、文件和临时想法都先放进 `01-收件箱/`。你可以让 Agent 盘点、归类和提出建议，但它不会未经确认移动或删除原件。

## 项目：需要持续推进的事情

当材料对应一个要持续做、需要跟进进展的目标时，它会关联到 `03-项目/`。你可以在那里看到当前状态、阻塞和下一步。

## 知识：已经确认、以后还能使用的结论

值得长期保留的理解，会先成为知识草稿。你确认后，它才成为正式知识；如果后来出现相反证据，系统会先请你处理冲突，而不会悄悄覆盖旧结论。

## 阅读：和一份内容持续相处

书籍、长文或连续章节可以进入[[00-系统/模块/阅读|阅读模块]]。那里保留进度、批注和阅读洞察；洞察不会自动被当成正式知识。

## 归档：退出当前视野，不等于删除

结束或暂时不需要的内容可以归档，以后仍能回看。归档前会由你确认。
