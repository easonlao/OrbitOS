# Agent 协作

OrbitOS 支持不同 Agent 接着完成同一件事。你不需要在不同对话之间手动复制背景；需要交接时，直接说明要把事情交给另一位 Agent。

## 什么时候需要交接

- 一位 Agent 已完成调研，另一位要继续实现或审查。
- 事情需要不同能力配合，例如设计、实现和复核。
- 你想隔一段时间或换设备后继续，但不想重新说明全部背景。

只需要这样说：

```text
交给另一位 Agent 继续。
```

Agent 会自行整理已经完成什么、还需要你确认什么，以及接手后从哪里开始。

另一位 Agent 接入后，只需要说：

```text
获取交接工作。
```

系统会找到适合它接手的任务，并先说明当前目标、已完成、待确认和第一步。你不需要告诉它交接文件放在哪里。

## 交接后你会得到什么

一份好的交接会让下一位 Agent 知道：

- 当前结论是什么，以及依据来自哪里。
- 已经完成什么，哪些仍未完成。
- 有什么风险或待确认事项。
- 接下来最小的一步是什么。

项目本身的进展仍留在项目页面；交接只负责让协作不断线。

## 让协作更贴近你

如果你希望 Agent 更理解你的协作节奏、表达偏好或决策习惯，可以使用[[00-系统/模块/人物|人物模块]]。如果你希望跨会话少重复说明稳定背景，可以了解[[00-系统/模块/Hindsight|Hindsight]]。

复杂协作需要更稳的拆解、判断或审查时，Agent 可以使用 [[05-思考方法|思考方法]]为本轮任务匹配思考模型。
