# 术语表

本页用于按需解释正文中无法避免的 OrbitOS 系统术语，不需要从头阅读或提前记忆。项目确有专门业务或技术概念时可以使用自己的已确认术语表；普通项目管理语言不需要进入术语表。

- 点击正文中的术语链接会跳到对应解释。
- 开启 Obsidian 核心插件 **Page Preview** 后，悬停术语链接即可直接预览。
- `%% 注释 %%` 在阅读视图中不可见，因此系统不会用隐藏注释承载必要解释。
- 只有当前页面特有的补充才使用脚注；可复用解释统一放在这里。

## Agent

具体执行者，例如 Hermes、Nova、Codex 或 Claude Code。

## Agent Profile

某个 Agent 的接入档案，记录部署位置、运行限制、经验和踩坑。它不定义专家角色。

## Startup Sync

Agent 开始工作前的只读同步。用于确认身份、运行环境、当前状态和历史注意事项。

## Progress Sync

完成实质性工作后的同步。用于记录事实、校验结果，并刷新项目状态和时间线。

## Dashboard

用户查看系统当前状态的聚合页面，目前由 [[../02-时间线/今日|今日]] 承担。

## Event

机器事实记录，说明谁在什么时候、因为什么、做了什么。Event 不等于日记、知识或完整对话。

## Workflow

Agent 完成某类任务时遵循的执行步骤，例如收件箱盘点、进度同步和周回顾。

## Rule

Agent 必须遵守的稳定约束。个人经验不会自动成为公共规则。

## Schema

机器记录的结构约束，规定必填字段、字段类型和合法值。

## Validation

对 schema、目录、链接和生命周期执行的自动检查。

## Lifecycle

内容从原始输入、盘点、确认、处理到归档的状态变化。

## Queue

Agent 保存中间判断和待执行建议的机器队列，不是用户主要阅读入口。

## 当前知识

用户确认后位于 `04-知识/` 正式主题目录的综合理解。核心结论变化前必须回到草稿箱重新确认。

## Artifact

面向人阅读或复用的正式产物，例如文章、报告或知识卡片。

## Hindsight

可选的跨会话长期记忆系统。它可以增强 Recall 和 Retain，但不是 OrbitOS 的事实底座。

## Runtime

你实际使用的 OrbitOS 工作副本，包含本地内容和运行状态。

## Product Repo

用于维护并发布 OrbitOS 系统文件的 Git 仓库。Runtime 从这里拉取系统更新。
