# 目录说明

OrbitOS 根目录按内容从进入到沉淀的顺序排列。

| 目录 | 放什么 | 你什么时候看 |
|---|---|---|
| `00-系统/` | OrbitOS 使用说明 | 初次使用或不了解规则时 |
| `01-收件箱/` | 尚未处理的文字、链接、文件和材料 | 投递内容或确认盘点结果时 |
| `02-时间线/` | 今日、本周 | 每天优先查看 |
| `03-项目/` | 正在推进的具体工作 | 处理某个项目时 |
| `04-知识/` | 经确认的当前知识和草稿 | 查找长期可复用理解时 |
| `.orbitos/` | Agent 规则、工作流和机器记录 | 普通使用不需要查看 |

## 三个最常用入口

- `01-收件箱/00-粘贴.md`：随手投递内容。
- `02-时间线/今日.md`：看今日页。
- `02-时间线/今日.md` 里的需要确认事项：处理需要你决定的事项。

如果你平时只打开 3 个地方，优先就是这三个。其他目录大多是在你明确要看项目、知识或归档时再进入。

## 按需出现的目录

| 目录 | 由什么创建 | 说明 |
|---|---|---|
| `05-阅读/` | 阅读模块进入 `ready` | 书籍、文章的深读、批注、进度和个人洞察 |
| `99-归档/` | 首次确认归档对象时 | 不会在首次初始化时创建 |

`06-资源/` 与 `07-输出/` 保留编号但当前没有模块包，因此不应手动预建或让 Agent 自行创建。旧工作副本若已存在这些目录，只作为迁移前用户内容保留，不代表当前可继续使用的流转区域。需要了解模块何时会创建可见目录，见 [[10-模块与扩展|模块与扩展]]。

## 项目目录

每个项目至少应让你快速找到：

- `STATUS.md`：当前做到哪里、有什么阻塞，以及下一步从哪里继续。

`AGENTS.md` 是 Agent 的项目入口，普通阅读不需要打开。项目管理目录默认不创建 README；产品仓库或可独立使用模块可以保留面向用户的 README。

项目可以包含代码、应用、素材和中间文件，不需要套用知识区结构。

如果一项任务还没有合适现有项目，Agent 会先判断是否应该挂接到已有项目，或在得到你确认后新建项目。

如果项目同时包含本地管理材料和独立发布仓库，推荐使用：

```text
03-项目/{项目名}/
  AGENTS.md
  STATUS.md
  docs/
  repo/  # 可选，仅代码型项目需要
```

`docs/` 保存仍有效的专项补充；`repo/` 只在项目确实需要独立产品仓库时存在，也就是项目的 [[06-术语表#Product Repo|Product Repo]]。

当前项目协作边界也固定在这里：

- 项目目录继续承担人读项目空间，不复制另一套 `project / wiki / profile` 并列目录树。
- 项目内多 Agent 协作默认通过 handoff 承接，而不是把中间判断直接塞回项目状态或知识区。
- 项目 `STATUS.md` 继续只承担项目当前状态；handoff 负责协作链、接手动作和局部回退。

## 附属目录

- `00-系统/agents/`：已接入 Agent 的档案和看板，不属于主阅读顺序。
- `02-时间线/归档/`：旧周报等时间视图快照，不等于 `99-归档/`。

## 共享记忆怎么分层看

如果你是从“长期复用”角度找内容，可以按下面这层分工理解：

- `03-项目/`：项目当前状态、项目设计和项目级经验。
- `04-知识/`：已经确认、可以复用的知识结论。
- `05-阅读/`：阅读模块启用后，与书、文章和阅读现场绑定的进度、批注与个人洞察。
- Hindsight：只保留跨会话可召回的最小稳定摘要，不替代前两层主源。
