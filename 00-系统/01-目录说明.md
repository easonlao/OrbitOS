# 目录说明

OrbitOS 根目录按内容从进入到沉淀的顺序排列。

| 目录 | 放什么 | 你什么时候看 |
|---|---|---|
| `00-系统/` | OrbitOS 使用说明 | 初次使用或不了解规则时 |
| `01-收件箱/` | 尚未处理的文字、链接、文件和材料 | 投递内容或确认盘点结果时 |
| `02-时间线/` | 今日、本周 | 每天优先查看 |
| `03-项目/` | 正在推进的具体工作 | 处理某个项目时 |
| `04-知识/` | 经确认的当前知识和待确认草稿 | 查找长期可复用理解时 |
| `05-资源/` | 已识别、可被引用的图片、PDF 和资料 | 需要引用原始资料时 |
| `06-输出/` | 在 Obsidian 内形成的文章、报告等成品 | 阅读或发布成果时 |
| `99-归档/` | 已退出当前工作视野的完整对象 | 追溯旧内容时 |
| `.orbitos/` | Agent 规则、工作流和机器记录 | 普通使用不需要查看 |

## 三个最常用入口

- `01-收件箱/00-粘贴.md`：随手投递内容。
- `02-时间线/今日.md`：查看系统当前状态。
- `02-时间线/今日.md` 里的“当前待确认”：处理需要你决定的事项。

## 项目目录

每个项目至少应让你快速找到：

- `STATUS.md`：当前做到哪里、有什么阻塞，以及当前最多 3 项任务。
- `ROADMAP.md`：需要长期规划时显示目标、总体状态和完成条件清单；已验证条件标记日期，当前任务仍在 STATUS。

`AGENTS.md` 是 Agent 的项目入口，普通阅读不需要打开。项目管理目录默认不创建 README；产品仓库或可独立使用模块可以保留面向用户的 README。

项目可以包含代码、应用、素材和中间文件，不需要套用知识区结构。

如果项目同时包含本地管理材料和独立发布仓库，推荐使用：

```text
03-项目/{项目名}/
  AGENTS.md
  STATUS.md
  ROADMAP.md
  docs/
  repo/
```

`docs/` 保存仍有效的专项补充；`repo/` 保存真正发布的产品或代码仓库，也就是项目的 [[06-术语表#Product Repo|Product Repo]]。

## 附属目录

- `00-系统/agents/`：已接入 Agent 的档案和看板，不属于主阅读顺序。
- `02-时间线/归档/`：旧周报等时间视图快照，不等于 `99-归档/`。
