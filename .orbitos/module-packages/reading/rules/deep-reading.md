# Deep Reading Rule

本规则约束深读陪读过程中的交互边界、写入权限和完成语义。

## 交互边界

### 原文展示

- 每次展示不超过 1 个阅读单元的原文。
- 展示原文后必须附上引导语，说明当前单元在全章中的位置。
- 展示原文时只准备空骨架和本轮讨论目标；在用户反馈前不得预填 2-7 段内容。

### 提问规则

- 每次只问一个理解性问题，等用户回答后再问下一个。
- 问题类型从以下中选择：复述、追问、反驳、反例、观点碰撞。
- 不连续使用同一类型问题。
- 用户未回答时不推进到下一个问题。

### 暂停条件

满足以下任一条件时暂停深读：
- 用户明确说"停一下"或"先到这里"
- 讨论偏离当前单元主题超过两轮
- 连续两轮用户不回应
- 达到当前注意力预算上限

### 推进条件

满足以下任一条件时可推进到下一个单元：
- 用户明确说"继续"或"下一个"
- 当前单元的 2-7 段已填充完毕，且用户明确确认完成并允许推进
- 用户主动提出下一个单元的话题

## 写入权限

| 内容 | 写入位置 | 谁写 | 条件 |
|------|----------|------|------|
| 第 2-7 段 | `chNN/NN.md` | Agent | 讨论中渐进写入 |
| 批注 | `chNN/NN.notes.md` | Agent（脚本） | 用户明确要求时 |
| 第 8 段 | `chNN/NN.md` | 用户确认后由 Agent 原样或近原样记录 | 不得替用户补观点 |
| 第 9 段 | `chNN/NN.md` | 用户明确要求回看时由 Agent 记录 | 默认留白 |
| 章节提炼 | 独立产物 | Agent | 所有单元完成后 |
| 对话临时讨论 | 不持久化 | — | — |

## 完成语义

### 单元完成

单元的 2-7 段全部填充完毕且用户确认后，标记为 `unit-complete`：
- 更新 `progress.md` 中对应单元的 checkbox 为 `[x]`。
- 不标记章级完成。

### 章节提炼完成

所有单元标记为 `unit-complete` 后，生成章级总结性提炼：
- 提炼覆盖全章所有单元的理解。
- 提炼不写入任何单元文件。
- 标记章节为 `chapter-distilled`。

### 章节深读完成

章节提炼完成后，经独立审查确认：
- 无重大理解遗漏。
- 全章理解连贯。
- 标记章节为 `chapter-deep-read`。

**关键约束**：覆盖所有单元 ≠ 章节深读完成。必须显式完成章节提炼和审查。

在多单元章节的 `progress.md` 章节行上，只有同时写明 `chapter-deep-read` 和 `review: passed` 后，才允许使用 `[x]`；这两个标记代表章节提炼和独立审查已经完成。

## 禁止

- 不改写 `raw.md`。
- 不在一次对话中一次性填充所有 2-7 段。
- 不跳过用户反馈直接写入笔记。
- 不把对话中的临时讨论写入持久化文件。
- 不在未完成所有单元前标记章节深读完成。
- 不用章节级提炼覆盖单元级深读缺口。
