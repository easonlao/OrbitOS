# Deep Reading Workflow

深读陪读：围绕一个阅读单元的原文展开交互式理解，渐进填充 2-7 段笔记，最终完成单元深读。

## 前提

- 目标章已完成 `chapter-split`，至少有一个骨架态单元。
- 用户明确表示要继续阅读或深读某个单元。
- 已读取 `CONTEXT.md` 中的深读陪读协议。

## 流程

### 1. 定位当前单元

从 `progress.md` 读取章→单元两层进度，找到下一个待深读的单元。若用户指定了单元，直接定位。

### 2. 展示原文

- 从 `chNN/NN.md` 读取第一段（原文）。
- 每次展示不超过 1 个完整阅读单元的原文。
- 展示后附上引导语，说明当前单元在全章中的位置和主题。

### 3. 保持骨架并提出第一问

展示原文后不预填 2-7 段，只根据原文提出一个第一问。默认先让用户用自己的话复述，不提供完整解释或摘要替代用户阅读。

### 4. 根据反馈渐进生成内容

每轮用户反馈后，只生成与本轮讨论直接相关的 2-7 段内容：

- **第 2 段（字词注）**：识别单元中的关键术语，逐个解释。
- **第 3 段（白话直译）**：用现代汉语尽量贴字面翻译。
- **第 4 段（注家对照）**：有可靠对照材料时，区分原文、对照来源和 Agent 推断。
- **第 5 段（与上一单元的连接）**：说明与前文的承接关系。
- **第 6 段（三个值得停下来想一想的问题）**：生成启发性提问。
- **第 7 段（与生活的关联）**：把抽象内容拉回具体经验。

**生成规则**：不得在用户反馈前生成；不得在一次对话中一次性填充全部 2-7 段；每次写入后说明写入了哪一段以及依据。

### 5. 向用户提问与碰撞

展示原文和初步引导后，向用户提出理解性问题。问题类型从以下中选择：

- **复述**：让用户用自己的话重述核心内容。
- **追问**：对模糊或有歧义处深入追问。
- **反驳**：提出与原文相反的立场。
- **反例**：给出与原文论点矛盾的现实案例。
- **观点碰撞**：邀请用户表达个人立场。

每次只问一个问题，等用户回答后再问下一个。

当用户给出理解后，Agent 必须先检查其是否被原文支持，再进行至少一种有效挑战：指出遗漏、提出反例、给出相反立场或追问隐含前提。不同意时要区分原文事实、Agent 推断和用户立场，不用“你说得对”替代审查。

### 6. 回应用户反馈

根据用户的回应类型，调整讨论方向：

| 用户回应 | Agent 行为 | 写入 |
|----------|-----------|------|
| 复述准确 | 指出原文依据，并补充遗漏点或反例 | 补充到对应段 |
| 复述有偏差 | 指出偏差，引导修正 | 修正对应段 |
| 追问 | 提供更多上下文 | 补充到对应段 |
| 反驳 | 认真回应，不回避 | 写入批注或对应段 |
| 反例 | 评估反例影响 | 写入批注或对应段 |
| 观点碰撞 | 保持中立但不迎合，帮助理清立场 | 写入对应段或保留未决分歧 |
| 沉默/不清楚 | 换个角度重新提问 | 不写入 |

### 7. 暂停与推进判定

**暂停条件**（满足任一即暂停）：
- 用户明确说"停一下"或"先到这里"
- 讨论偏离当前单元主题超过两轮
- 连续两轮用户不回应
- 达到当前注意力预算上限

**推进条件**（满足任一可推进）：
- 用户明确说"继续"或"下一个"
- 当前单元的 2-7 段已填充完毕
- 用户主动提出下一个单元的话题

### 8. 单元完成判定

当 2-7 段全部填充完毕且用户确认后，标记单元为 `unit-complete`：
- 更新 `progress.md` 中对应单元的 checkbox 为 `[x]`。
- 不标记章级完成。

### 9. 章节提炼（所有单元完成后）

当章内所有单元都标记为 `unit-complete` 后：
- 生成章级总结性提炼，覆盖全章所有单元的理解。
- 章节提炼不写入任何单元文件，是独立产物。
- 标记章节为 `chapter-distilled`。

### 10. 章节深读审查

章节提炼完成后，由独立子 agent 或用户确认：
- 审查是否有重大理解遗漏。
- 确认全章理解的连贯性。
- 审查通过后标记章节为 `chapter-deep-read`。
- 多单元章节的 `progress.md` 章节行必须同时包含 `chapter-deep-read` 和 `review: passed`，再改为 `[x]`。

## 持久化

- 第 2-7 段内容在讨论中渐进写入 `chNN/NN.md`。
- 批注在用户明确要求时写入 `NN.notes.md`。
- 对话中的临时讨论不持久化。
- 第 8 段只能在用户确认其理解/疑问后由 Agent 记录；第 9 段默认留白，只在用户明确要求回看时记录。

## 禁止

- 不改写 `raw.md`。
- 不在一次对话中一次性填充所有段。
- 不跳过用户反馈直接写入笔记。
- 不把对话中的临时讨论写入持久化文件。
- 不在未完成所有单元前标记章节深读完成。
