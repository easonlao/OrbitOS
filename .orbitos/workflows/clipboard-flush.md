# Clipboard Flush Workflow

`01-收件箱/00-粘贴.md` 是持续复用的快速输入位，不是可入库原件。本 workflow 把用户确认的粘贴条目物化为独立原件，防止同一内容被反复处理。

## 触发条件

- 用户说“整理粘贴内容”“处理粘贴内容”或等价表达。

## 流程

1. 只读取 `00-粘贴.md`，按链接、段落或用户指定范围列出候选条目；不把整个剪贴板当成一个原件。
2. 为每个候选条目给出建议名称、原始内容和可能去向；此时不创建知识卡片、不移动文件、不删除粘贴内容。
3. 明确向用户说明确认动作的效果：
   - 为选中的条目创建独立原件到 `01-收件箱/`。
   - 仅在原件创建成功后，从 `00-粘贴.md` 删除相同的已消费条目。
4. 用户确认后，逐条创建原件，保留原始 URL、摘录或文字；文件名使用可读的原始主题，不把多个无关条目混成一个文件。
5. 只有所有已确认条目均已物化且可读取时，才删除剪贴板中的对应原文；不添加“已处理”标记，不把剪贴板变成第二份处理记录。
6. 把新原件交给 `inbox-triage.md`；得到用户确认后，再由 `inbox-ingest.md` 移入 `已入库/` 并登记 batch。
7. 执行 Validate Sync 与 Progress Sync。

## 边界

- 图片、PDF 和其他附件应直接放入 `01-收件箱/`，它们本身就是原件；不要先嵌进剪贴板再复制。
- 物化失败、文件名冲突或用户没有确认时，保留全部粘贴原文，不做局部清理。
- `00-粘贴.md` 本身永不移动、永不登记 batch、永不作为知识来源。
- 删除已消费条目属于用户内容修改，必须由第 3 步的明确确认覆盖。
