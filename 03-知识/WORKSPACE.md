# 03-知识 工作区规范

## 用途

沉淀已经处理过的知识卡片、主题笔记、学科笔记和概念页。

## 允许

- 知识卡片
- 主题笔记
- 学科笔记
- 概念解释

## 禁止

- 原始网页全文
- 项目执行文件
- 发布成品

## 命名

使用 `YYYYMMDD_主题.md`。目录内部保持卡片扁平，不再建立任何二级主题目录。

## Frontmatter

`workspace` 必须为 `03-知识`。知识卡片的 `type` 使用 `note` 或 `card`；领域地图（MAP）的 `type` 使用 `roadmap`。
除了保留现有的 9 字段 Frontmatter 之外，允许扩展以下字段以支持卡片关联（非强制历史文件补齐）：
- `aliases`: 别名列表，方便在双向链接时自动匹配。
- `up`: 指向对应的上位领域 MAP 链接（如 `[[00_MAP_金融交易]]`）。
- `related`: 相关卡片链接列表。

## 子 Skill

使用 `workspace-knowledge`。

## 推荐子目录结构 (六大长期领域)

`03-知识` 的一级目录只表达长期领域，不表达细分主题。所有新卡片和知识整理进入以下六个收敛的长期领域目录中：

- `AI与Agent/` - 大模型理论、Agent 工作流、提示词工程、多 Agent 协作、AI 工程开发与应用实践。
- `软件与系统/` - 编程语言（Python、JS 等）、系统架构、软件设计模式、基础算法、开发工具使用技巧。
- `金融交易/` - 市场微观结构与做市商理论、仓位与风险管理、交易策略、基本面分析与交易心理学。
- `认知与方法论/` - 个人知识管理系统（PKM）最佳实践、学习与思考方法论、认知神经科学等。
- `阅读与书籍/` - 精读笔记、书籍卡片与长篇读书心得。
- `本地基础设施/` - 本地局域网设备拓扑、存储同步架构（群晖 NAS, SMB, Synology Drive 等）、物理或虚拟化运行环境部署与网络基础设施。

> [!NOTE]
> 历史旧细分目录（如 `AI工程/`、`AI/`、`开发/`、`工具与工作流/` 等）已标记为**历史旧目录**。除非用户明确要求，Agent 不得在这些目录中新建任何卡片。现有内容将在后续分批迁移。

## 结构与导航规则 (MAP 模式)

1. **领域内保持扁平**：长期领域目录内部不再划分子目录，知识卡片平铺存放。
2. **MAP 入口控制**：每个长期领域目录根部必须包含一个 `00_MAP_领域名.md` 索引地图（例如 `00_MAP_金融交易.md`）。
3. **阅读路径解耦**：物理目录不承担阅读路径职责。阅读路径、概念层级和顺序完全由 MAP 地图、双链和 Tag 标签承载。
4. **卡片新增机制**：新建知识卡片时，优先写入对应的长期领域目录，并检查/更新对应的 `00_MAP` 文件，确保新卡片不成为孤立卡片。

## 原子卡片规范

1. **单一概念原则**：一张卡片只表达一个明确的概念、模型、方法、经验或判断。
2. **多重归属引用**：卡片是原子化的，可以被多个 MAP 引用，但物理上只存在一个文件，严禁因多重归属而复制或拆分文件。
3. **二级标题分块**：卡片正文应使用清晰的二级标题，方便 Agent 进行分块读取和精细定位。
4. **标准关联区**：卡片底部建议包含“关联”区，以便于形成网状链接：
   - **上位地图**：`up: [[00_MAP_领域名]]`
   - **相关概念**：`related: [[相关卡片名]]`
   - **应用场景**：主要应用场景
   - **反例 / 风险**：局限性、不适用场景或常见陷阱
