# OrbitOS

> 一个以 Obsidian 为阅读界面、以 Agent 为执行伙伴的本地工作系统。

OrbitOS 把零散材料、持续项目和已确认的知识放进同一套可继续的工作节奏：你表达目标、查看结果并确认关键决定；Agent 负责整理、推进和留下可回看的记录。

## 它解决什么问题

- 新材料有唯一入口，不会一开始就散落到项目或知识库。
- 正在推进的事情能看到当前进展、阻塞和下一步。
- 已确认的结论可以沉淀为知识，未确认内容保留为草稿。
- 换 Agent、换设备或隔一段时间后，仍能从当前状态继续。
- 重要的移动、删除、归档和定稿始终由你确认。

## 核心能力

- **收件箱**：先接住文字、链接、文件和想法，再决定去向。
- **项目**：持续推进具体目标，并保留当前状态。
- **知识**：沉淀已确认、以后可复用的理解。
- **今日页**：集中查看现在的进展、待确认事项和可继续入口。
- **按需模块**：阅读、人物、记忆、工程、自动化和多 Agent 协作只在真正需要时启用。

## 安装

1. 克隆仓库：

   ```powershell
   git clone https://github.com/easonlao/OrbitOS.git
   ```

2. 用 Obsidian 打开 `OrbitOS` 文件夹。
3. 初始化本地工作副本：

   ```bash
   python .orbitos/scripts/init-runtime.py
   ```

4. 使用任意 Agent 时，让它从 `AGENTS.md` 开始。

初始化不会覆盖已有内容。之后把材料放进 `01-收件箱/`，再直接描述你希望 Agent 完成什么。

## 第一次使用

1. 把文字、链接或资料放进 `01-收件箱/`。
2. 对 Agent 说：`请先看看收件箱里有什么，给我处理建议，不要移动文件。`
3. 打开 `02-时间线/今日.md`，查看进展、待确认事项和下一步。
