# OrbitOS

> 一个 **Agent-native** 个人知识库操作系统。打开 AI Agent，直接说话就能用。

---

## 快速开始

```bash
git clone https://github.com/easonlao/OrbitOS.git my-vault
```

用 Obsidian 打开 `my-vault`，在设置 → 第三方插件中启用 **OrbitOS Dashboard**。

打开 AI Agent，说"帮我初始化这个知识库"——完成。

**日常使用：** 直接说你想做什么。不确定说什么？说"帮助"。

---

## 核心流程

一个想法从诞生到落地，六个阶段：

```
模糊想法 → 打磨 → 验证 → 写清楚 → 对齐 → 执行 → 复盘
```

| 阶段 | 你说 | 会发生什么 |
|------|------|-----------|
| ① 打磨 | "头脑风暴" | AI 和你来回讨论，质疑假设，帮你把模糊想法变清晰 |
| ② 验证 | "启动项目" | AI 给你的想法打分（满分30），找出核心假设和致命缺陷，告诉你值不值得做 |
| ③ 写清楚 | "写PRD" | AI 追问你问题（用户是谁、成功怎么衡量、不做什么），然后生成需求文档 |
| ④ 对齐 | "grill me" | AI 一个问题一个问题追着你问，确保你没遗漏盲点 |
| ⑤ 执行 | "研究一下" | AI 先给你看研究计划，你确认后它去搜索、阅读、整理成笔记 |
| ⑥ 复盘 | "周报" | AI 扫描你本周的所有记录，生成一份结构化总结 |

**不是每次都要走完。** 小想法可能只需要 brainstorm → capture。大项目走全部。

## 工作区

AI 根据内容自动归类，你不用手动建文件夹：

| 工作区 | 放什么 |
|--------|--------|
| 01-收件箱 | 临时想法、未分类内容（AI 会整理） |
| 02-日记 | 工作日志、反思、复盘（AI 会创建和追加） |
| 03-知识 | 知识卡片、研究笔记（AI 会分类存放） |
| 04-项目 | 项目文档、PRD（AI 会按类别归档） |
| 05-资源 | 参考资料、模板（你手动放） |
| 06-输出 | 可发布内容（AI 会管理状态） |
| 99-归档 | 已完成/废弃内容（AI 会自动归档） |

## 设计理念

- **Agent-native**：所有操作由 AI Agent 驱动，Skills 内置在 vault 内
- **自描述**：`.orbit/workspace-index.yaml` 即代表"这里是知识库"
- **渐进式加载**：Skill 按场景按需加载，不浪费 token
- **先记下来，再整理**：收件箱让你永不丢失灵感
- **连接比分类重要**：双向链接构建灵活可查的知识图谱
- **每日节奏**：工作日志锚定一切，为工作和思考建立时间线

---

## 三个关键机制

**渐进式加载：** AI 不会一次读完所有技能。先判断你在做什么，只加载需要的。

**确认点：** 关键操作前 AI 会问你确认——研究方向、压力测试结果、PRD内容。不会擅自执行。

**自动化：** git commit 自动记录到工作日志 / 每天8点自动创建工作日志 / 每2小时检查日志是否需要补充。

## 目录结构

```
.orbit/                   ← vault 根锚点 + schema 规范
00-系统/
  Skills/                 ← 全部 Skill 的 canonical 位置
  规范/                   ← 命名、Frontmatter、路由、审计规范
  运行时/                 ← hooks、crontab、自动化模板
01-收件箱/                ← 所有未分类内容的入口
02-日记/                  ← 工作日志、反思、复盘、人际事件
03-知识/                  ← 知识卡片、主题笔记、长期沉淀
04-项目/                  ← 项目文件，按意图分类
05-资源/                  ← 图片、附件、参考资料
06-输出/                  ← 对外发布内容
99-归档/                  ← 已归档内容
```
